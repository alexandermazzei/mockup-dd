API Documentation

API Version v2.0
This documentation is for the latest API version (v2.0). For older versions, please see the version history. Support for v1.x will be discontinued on December 31, 2025.
Introduction

The DueDiligenceAI API provides programmatic access to all the features of our platform, allowing you to integrate due diligence workflows into your own systems and applications. With our API, you can automate document collection, analyze financial data, generate reports, and more.

Base URL

All API requests should be made to the following base URL:

https://api.duediligenceai.com/v2/

Content Types

Our API accepts and returns JSON data. All requests should include the following header:

Content-Type: application/json

Request Examples

Throughout this documentation, we'll use curl commands to demonstrate API calls:

Example Request
curl -X GET "https://api.duediligenceai.com/v2/documents" \ -H "Authorization: Bearer YOUR_API_KEY" \ -H "Content-Type: application/json"
Authentication

DueDiligenceAI API uses API keys to authenticate requests. You can view and manage your API keys in the Developer Dashboard.

API Key Security
Keep your API keys secure and never share them in publicly accessible areas such as GitHub, client-side code, or in plain text. API keys should only be stored on your servers.

Bearer Authentication

All API requests must include your API key in the Authorization header using Bearer authentication:

Authorization: Bearer YOUR_API_KEY

API Key Types

DueDiligenceAI offers two types of API keys:

Key Type Description Permissions
Production Keys For use in production environments Full access to all API features based on your subscription plan
Test Keys For development and testing Same access as production keys but connect to a sandbox environment with test data

Creating API Keys

To create a new API key:

  1. Navigate to the Developer Dashboard
  2. Click on API Keys in the sidebar
  3. Click Create New API Key
  4. Select the key type (Production or Test)
  5. Add a description for the key
  6. Click Create
Important Note
API keys are only displayed once at creation time. Make sure to copy and securely store your API key, as you won't be able to view it again.

API Key Permissions

Access to specific API endpoints is determined by your subscription plan and the permissions granted to your API key. If you receive a 403 Forbidden error, your API key may not have the necessary permissions to access the requested endpoint.

Rate Limiting

To ensure the stability and availability of our API for all users, we implement rate limiting based on your subscription plan. Rate limits are applied per API key.

Rate Limit Headers

All API responses include headers that provide information about your current rate limit status:

Header Description
X-RateLimit-Limit The maximum number of requests allowed in a time window
X-RateLimit-Remaining The number of requests remaining in the current time window
X-RateLimit-Reset The time when the current rate limit window resets (Unix timestamp)

Rate Limit Tiers

Rate limits vary based on your subscription plan:

Plan Rate Limit Burst Limit
Basic 100 requests per minute 150 requests
Professional 300 requests per minute 450 requests
Enterprise 1000 requests per minute 1500 requests
Custom Contact sales for custom limits Contact sales

Rate Limit Exceeded

If you exceed your rate limit, you will receive a 429 Too Many Requests response. The response will include the X-RateLimit-Reset header indicating when you can resume making requests.

Rate Limit Exceeded Response
{ "error": { "code": "rate_limit_exceeded", "message": "Rate limit exceeded. Please retry after 60 seconds.", "retry_after": 60 } }

Best Practices

To avoid rate limiting issues, we recommend the following best practices:

  • Implement exponential backoff when retrying failed requests
  • Cache responses when appropriate to reduce the number of API calls
  • Batch API calls when possible instead of making multiple individual requests
  • Monitor your usage with the rate limit headers
Errors

DueDiligenceAI uses conventional HTTP response codes to indicate the success or failure of an API request. In general, codes in the 2xx range indicate success, codes in the 4xx range indicate an error that resulted from the provided information (e.g., missing parameters, invalid values), and codes in the 5xx range indicate an error with DueDiligenceAI's servers.

HTTP Status Codes

200 OK - Request succeeded
201 Created - Resource created
204 No Content - Request succeeded with no response body
301 Moved Permanently - Resource has moved
400 Bad Request - Invalid request format
401 Unauthorized - Authentication failed
403 Forbidden - Insufficient permissions
404 Not Found - Resource not found
409 Conflict - Resource conflict
422 Unprocessable Entity - Validation failed
429 Too Many Requests - Rate limit exceeded
500 Internal Server Error - Server error
503 Service Unavailable - Server unavailable

Error Object

When an error occurs, the API returns an error object with the following structure:

Error Response Format
{ "error": { "code": "error_code", "message": "A human-readable message providing more details about the error.", "details": { // Additional error details specific to the error type (optional) }, "request_id": "request-id-for-support" } }

Common Error Codes

Below are some common error codes you might encounter when using the API:

Error Code Description HTTP Status
authentication_failed Invalid API key provided 401
permission_denied The API key doesn't have permission to perform the request 403
resource_not_found The requested resource does not exist 404
validation_error The request parameters failed validation 422
rate_limit_exceeded Request rate limit exceeded 429
internal_error An unexpected error occurred on the server 500

Validation Errors

For validation errors (HTTP 422), the response includes additional details about the validation failures:

Validation Error Example
{ "error": { "code": "validation_error", "message": "The given data is invalid.", "details": { "project_name": [ "The project name field is required." ], "due_date": [ "The due date must be a valid date." ] }, "request_id": "req_1a2b3c4d5e6f" } }

Error Handling Best Practices

When handling errors from the API, we recommend the following best practices:

  • Always check the HTTP status code and error code in your API client
  • Implement appropriate retry logic with exponential backoff for 429 and 5xx errors
  • Log the request_id for debugging and support purposes
  • Handle validation errors by showing appropriate feedback to users

Core Resources

Documents

The Documents API lets you manage documents in your due diligence projects. You can upload, list, search, and categorize documents.

GET /v2/documents

Returns a list of documents in your account. The documents are returned sorted by creation date, with the most recently created documents appearing first.

Query Parameters

Parameter Type Description
project_id string optional Filter documents by project ID
category string optional Filter documents by category (e.g. 'financial', 'legal', 'corporate')
status string optional Filter documents by status (e.g. 'pending', 'processed', 'failed')
limit integer optional Limit the number of documents returned (default: 20, max: 100)
offset integer optional Offset for pagination (default: 0)

Example Request

List Documents in a Project
curl -X GET "https://api.duediligenceai.com/v2/documents?project_id=proj_1a2b3c4d&limit=10" \ -H "Authorization: Bearer YOUR_API_KEY" \ -H "Content-Type: application/json"

POST /v2/documents

Upload a new document to a project. The document will be processed asynchronously, and its status will initially be set to 'pending'.

Request Parameters

Parameter Type Description
project_id string required The ID of the project to upload the document to
file file required The file to upload (multipart/form-data)
category string optional Document category (e.g. 'financial', 'legal', 'corporate')
metadata object optional Additional metadata for the document

Example Request

Upload a Document
curl -X POST "https://api.duediligenceai.com/v2/documents" \ -H "Authorization: Bearer YOUR_API_KEY" \ -F "project_id=proj_1a2b3c4d" \ -F "category=financial" \ -F "file=@/path/to/Financial_Statement_2024.pdf"

Example Response

201 Created
{ "data": { "id": "doc_3c4d5e6f", "project_id": "proj_1a2b3c4d", "name": "Financial_Statement_2024.pdf", "file_type": "pdf", "file_size": 2548725, "category": "financial", "status": "pending", "uploaded_by": "user_2b3c4d5e", "created_at": "2024-05-07T10:15:32Z", "updated_at": "2024-05-07T10:15:32Z" } }
